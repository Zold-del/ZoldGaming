 POURQUOI CE CHANGEMENT ?
═══════════════════════════════════════════════════════════════

Sans "/blockdrop", MongoDB se connecte à la base "test" par défaut.
Vos collections (users, scores, tokenblacklists) sont dans "blockdrop".

C'est pourquoi les requêtes d'authentification échouaient (401 errors) :
- Le serveur cherchait les users dans "test" (vide)
- Les vrais users sont dans "blockdrop"
